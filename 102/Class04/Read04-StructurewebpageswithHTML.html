HTML Notes

HyperText Markup Language (HTML) - code used 2 structure web pgs and their content
Elements - enclose/wrap arnd parts of code 2 make it appear or act a certain way
Tags - makes words or images hyperlink 2 somewhere else, italicizes words, make font bigger/smaller etc My cat is very grumpy 2 seperate line specify it's a paragraph by enclosing in paragraph tags: <p>My cat is very grumpy</p> Opening tag - Name of element wrapped in angle brackets, states where element begins/starts 2 take effect
Closing tag - Same as opening tag, + forward slash before element name, states where element ends
Content - Content within element, btwn 2 tags
Element - Opening + closing tags + content
Attributes - Extra info abt element, doesn't appear in content
Nesting elements - Elements inside other elements
Void elements - Elements tht have no content <p class="editorâ€”note">My cat is very grumpy</p> *Class* is the attribute name & *editor-note* is the value. The *class* attribute allows u 2 give the element a non-unique identifier that can be used to target it (and any other elements with the same *class* value) with style information & other things. Some attributes have no value, such as *required*.

Attributes that set a value always have:

1. A space between it & element name (or previous attribute, if element has mtpl attributes).
2. Attribute name followed by =.

Attribute value wrapped in "&". <p>My cat is <strong>very</strong> grumpy.</p> we wrapped "very" in a <strong> element, meaning the word will be strongly emphasized, make sure elements are properly nested, <p> opened first then <strong> followed so <strong> must close first, then <p> last, elements must open & close correctly so they r clearly inside/outside <img src="images/firefox-icon.png" alt="My test image" /> This contains 2 attributes no closing </img> tag & no inner content, bc an image element doesn't wrap content 2 affect it, It embeds an image in the HTML page where it appears.
