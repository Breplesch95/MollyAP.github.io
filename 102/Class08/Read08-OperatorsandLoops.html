Expressions and operators

expression - a valid unit of code that resolves to a value,
There are two types of expressions, 1s with side effects like assigning values & 1s that purely evaluate.

x = 7 is an example of the 1st 1, it uses the = operator 2 assign the value seven to the variable x. The expression itself evaluates to 7

3 + 4 is an example of the 2nd 1, it uses the + operator to add 3 and 4 together and produces a value of 7, but if it's not eventually part of a bigger construct (like a variable declaration ex const z = 3 + 4), its results will immediately b discarded, usually this is a programmer mistake because the evaluation doesn't produce any effects.

ALL COMPLEX EXPRESSIONS ARE JOINED BY OPERATORS LIKE = AND +!

Types of operators:

Assignment operators - assigns a value to its left operand based on the value of its right operand

Comparison operators - compares its operands & returns a logical value based on whether the comparison is true

operators - takes numerical values (either literals or variables) as their operands & returns single numerical value

Bitwise operators - treats their operands as a set of 32 bits (zeros and ones), rather than as decimal, hexadecimal, or octal numbers

Logical operators - typically used with Boolean (logical) values, when they are, they return a Boolean value

BigInt operators - represent numeric values that r 2 large 2 be represented by the primitive

String operators - In addition 2 the comparison operators, the concatenation operator (+) concatenates two string values together, returning another string that is the union of the two operand strings

Conditional (ternary) operator - the only JavaScript operator that takes 3 operands. The operator can have one of two values based on a condition

Comma operator - evaluates both of its operands and returns the value of the last operand

Unary operators - an operation with only one operand

Relational operators - compares its operands and returns a Boolean value based on whether the comparison is true.

These operators join operands either formed by a higher-precedence operator or 1 of the basic expressions, the precedence of operators determines the order they get applied when evaluating an expression for example: const x = 1 + 2 * 3; const y = 2 * 3 + 1; Despite * & + coming in different orders both would result in 7 because * has precedence over +, so the * joined expression will always be evaluated 1st. You can override operator precedence by using parentheses (which creates a grouped expression)

JavaScript has binary, unary, and one special ternary operator, the conditional operator, a binary operator needs two operands, one before the operator and one after the operator: operand1 operator operand2 For example, 3 + 4 or x * y, this form is called an infix binary operator, because the operator is placed between two operands, all binary operators in JavaScript r infix. A unary operator requires a single operand, either before or after the operator: operator operand operand operator For example, x++ or ++x the operator operand form is called a prefix unary operator, & the operand operator form is called a postfix unary operator, ++ and -- are the only postfix operators in JavaScript â€” all other operators, like !, typeof, etc. are prefix.
