Remotes

Git will automatically name a cloned server “origin”
& your local branch “master” when u clone repositories.

The git remote cmd shows short names,
like “origin,” of all specified remote handles
git remote -v, shows all remote URLs next 2
corresponding short names
git remote add shortname url creates a new remote Git
repository with a short name
$ git remote
origin
$ git remote add js https://github.com/janesmith/project1
$ git remote -v
origin https://github.com/johndoe/project1 (fetch)
origin https://github.com/johndoe/project1 (push)
js https://github.com/janesmith/project1 (fetch)
js https://github.com/janesmith/project1 (push)
these remote & short names allows u 2 use shortnames 4 Git collaboration

Pushing

 git fetch [remote-name]
pulls data that u don’t have from a remote project
git fetch origin pulls down any new changes pushed
2 the server since u cloned or last fetched from it.
git fetch only pulls new data 2 a local repository
it does not merge changes with/modify your local work.
git push [remote-name][branch-name] pushes ur changes “upstream” 4 sharing
Example:
$ git push origin master
This cmd pushes committed changes from ur local “master” branch upstream 2 “origin” server

Renaming/Removing Remotes

 git remote rename renames a remote’s short name
$ git remote rename js jane
$ git remote
origin
jane

the remote’s short name was changed from js 2 Jane
The command git remote lists existing remotes which
jane is now one of, the rename action alters names of
remote branches 2, js/master changes 2 jane/master
